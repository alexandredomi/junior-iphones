Próximos passos - Junior iPhones

1) Criar projeto Firebase
   - Acesse https://console.firebase.google.com e crie um novo projeto (ex: junior-iphones)
   - Desative Google Analytics se preferir

2) Registrar app Web
   - Dentro do projeto: Adicionar app > Web (</>) > nome (site-junior-iphones)
   - Copiar as credenciais (apiKey, authDomain, projectId, appId, etc.)
   - Colar essas credenciais no `index.html` nas chaves do objeto `firebaseConfig`

3) Ativar Authentication (E-mail/Senha)
   - Firebase Console > Authentication > Métodos de login > ativar E-mail/senha
   - Criar o usuário administrador manualmente em Usuários (use um e-mail e senha seguros)

4) Criar Firestore
   - Firebase Console > Firestore Database > Criar banco de dados
   - Iniciar em modo Teste (para desenvolvimento), escolher região

5) Regras básicas de Firestore (inicial)
   - Use regras que permitam leitura pública e escrita somente por usuários autenticados:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /products/{productId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}

6) Testar o painel Admin localmente
   - Abrir `index.html` no navegador (ou servir via Live Server)
   - Clicar em "Admin" e entrar com o usuário criado
   - Adicionar um produto: verificar se aparece na lista pública

7) Popular catálogo inicial (opções)
   - Usar o painel Admin para adicionar produtos manualmente
   - Ou importar via script usando Admin SDK (opcional, para muitos produtos)

8) Configurar Firebase Hosting (opcional)
   - Instalar Firebase CLI: `npm install -g firebase-tools`
   - Login: `firebase login`
   - Inicializar: `firebase init` (selecionar Hosting e Firestore se quiser deploy integrado)
   - Deploy: `firebase deploy --only hosting`

9) Regras e segurança para produção
   - Trocar regras de Teste por regras restritivas antes de abrir ao público
   - Habilitar billing se usar funcionalidades pagas ou para limites maiores

10) Backups & monitoração
   - Planejar exportação periódica dos dados (Cloud Functions ou scripts)
   - Ativar alertas e monitoramento (Stackdriver / Firebase Alerts)

11) Próximos ajustes UX/Funcs
   - Validar imagens (hosted em Storage ou CDN)
   - Melhorar validações do formulário Admin
   - Adicionar logs de atividade (quem editou, quando)

Se quiser, faço a configuração do projeto Firebase passo a passo com você (criar projeto, colar config, criar usuário admin e ajustar regras).
